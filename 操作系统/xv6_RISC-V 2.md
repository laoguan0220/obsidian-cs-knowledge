## 一、核心执行流程（纯文本）

1. **硬件启动**：QEMU 加载内核镜像到 `0x80000000`
2. **机器模式入口**：`entry.S` → 初始化 CPU 栈 → 跳转 `start()`
3. **监督者模式切换**：`start.c` → 配置模式 / 中断 / 内存 / 定时器 → 跳转 `main()`
4. **内核初始化**：`main.c` → 初始化所有子系统 → 所有 CPU 进入调度器
5. **进程调度**：`proc.c` 调度器 → 选取可运行进程
6. **用户态启动**：`init.c` 首个进程 → 启动 Shell
7. **命令执行**：`sh.c` Shell → 运行各类用户程序

---

## 二、关键文件阅读顺序（推荐优先级）

1. `kernel/entry.S` —— 启动入口（机器模式）
2. `kernel/start.c` —— 切换至监督者模式
3. `kernel/main.c` —— 内核初始化主函数
4. `kernel/proc.c` —— 进程管理 + 调度器核心
5. `user/init.c` —— 第一个用户进程
6. `user/sh.c` —— Shell 命令行解释器
7. `kernel/syscall.c` —— 系统调用总入口
8. `user/usys.S` —— 用户态系统调用汇编接口

---

## 三、main.c 内核子系统初始化清单

- `kalloc.c`：物理内存分配器
- `vm.c`：虚拟内存管理
- `trap.c`：异常、中断、陷阱处理
- `proc.c`：进程创建、切换、管理
- `bio.c`：块设备缓存
- `fs.c`：文件系统核心逻辑
- `file.c`：文件描述符管理
- `virtio_disk.c`：虚拟磁盘驱动

---

## 四、常用用户程序

- `cat.c`：查看文件内容
- `ls.c`：列出目录文件
- `echo.c`：打印字符串
- `grep.c`：文本匹配搜索
- `wc.c`：统计行数 / 单词数 / 字符数

---

## 五、构建流程（Makefile 逻辑）

1. 编译内核：`kernel/*.c`/`*.S` → 目标文件 `.o`
2. 链接内核：`kernel.ld` 链接脚本 → 生成 `kernel/kernel`
3. 编译用户程序：`user/*.c` → 可执行程序
4. 构建文件系统：`mkfs` 工具 → 生成 `fs.img`（包含所有用户程序）

---

## 六、系统调用标准路径

plaintext

```
用户程序 → usys.S（用户态入口） → syscall.c（内核分发） → 具体系统调用函数
```

---

### 总结

1. 整体流程：**硬件启动 → 内核初始化 → 调度器 → init 进程 → Shell → 用户程序**
2. 阅读顺序：从汇编入口`entry.S`开始，按启动→初始化→调度→用户态的顺序学习最清晰
3. 核心模块：进程管理`proc.c`、虚拟内存`vm.c`、中断`trap.c`、系统调用`syscall.c`是理解重点
