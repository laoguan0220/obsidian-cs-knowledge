### 二、QEMU 的两种启动模式

#### 模式 A：正常启动（你现在的场景）

**QEMU 完整模拟 BIOS 执行 1~5 步**

text

```
1. CPU reset
2. 进入 16 位实模式
3. 执行 BIOS（固件代码）
4. 初始化硬件（内存、磁盘等）
5. 读磁盘第 0 扇区（MBR）→ 装入 0x7C00
6. CPU 跳转到 0x7C00 执行
```

**你用 GDB 调试看到的就是这个流程！**

- 初始 `CS:IP` 指向 `0xF000:0xFFF0`（BIOS 入口地址）
- 你可以单步跟踪 BIOS 代码执行
- 最终观察到 CPU 跳转到 `0000:7C00`

#### 模式 B：直接加载内核（xv6 使用的模式）

**QEMU **不执行 BIOS**，直接加载内核**

bash

运行

```
qemu-system-riscv64 -kernel kernel/kernel
```

- 这才是**真正跳过 1~5 步**
- QEMU 直接将内核二进制文件加载到内存指定地址
- 强制 CPU 从内核入口开始执行
- **此模式下，你看不到 BIOS 的执行过程**

### 四、最终核心流程（你看到的真实步骤）

1. **CPU reset**：CPU 上电 / 复位，强制进入 16 位实模式
2. **执行 BIOS**：CS:IP 指向 `0xF000:0xFFF0`，开始执行固件代码
3. **BIOS 自检与初始化**：完成硬件初始化（内存、磁盘等）
4. **读取 MBR**：BIOS 从磁盘第 0 扇区读取 512 字节到内存 `0x7C00`
5. **跳转执行**：BIOS 将 CS:IP 设置为 `0000:7C00`，将执行权交给你的 MBR 代码
6. **运行你的程序**：执行 `cli`, `out`, 填充等逻辑
