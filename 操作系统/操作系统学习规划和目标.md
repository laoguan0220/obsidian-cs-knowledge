1) 学习目标（你学成后能做什么）

2) 先决知识
- C 语言（指针、内存模型、make、gcc）——必要

- 数据结构与算法（链表、哈希表、栈、队列、树、并查集）
- 基本的汇编（x86-64 或 RISC-V）有助于理解上下文切换和中断

3) 核心主题与每个主题的“学法”
- 进程/线程与调度
  - 理论：进程/线程概念、上下文切换、调度策略（FCFS、RR、SJF、优先级、多级反馈队列）
  - 实践：在 xv6/简单内核实��上下文切换与调度；在用户态实现协程/线程库
- 并发与同步
  - 理论：锁/自旋锁/互斥、条件变量、信号量、无锁编程、死锁（产生条件、避免、检测、恢复）
  - 实践：实现自旋锁、互斥锁；用多线程模拟竞态并用工具验证（helgrind/TSAN）
- 内存管理
  - 理论：地址空间、分页、TLB、页面置换算法（LRU、Clock）、内存分配器（伙伴、slab）
  - 实践：在 xv6/rust-os 添加分页或实现简单 malloc；实现页面置换模拟器
- 文件系统
  - 理论：文件/目录抽象、inode、块分配、日志（journaling）
  - 实践：实现一个简单的日志式文件系统或在 xv6 上扩展文件操作
- I/O 与设备驱动
  - 理论：中断驱动、DMA、字符/块设备接口
  - 实践：写一个简单的 Linux 内核模块（字符设备），在 /dev 下测试
- 虚拟化/安全/性能
  - 理论：虚拟机监控器、容器、权限模型、调试/追踪方法
  - 实践：用 QEMU/KVM 运行多系统、用 perf/oprofile 做性能分析

4) 推荐书籍与课程（先中文/后英文）
- Operating Systems: Three Easy Pieces (OSTEP) — 免费、实践导向，强烈推荐（有中文翻译）
-《现代操作系统》（Andrew S. Tanenbaum）— 概念清晰，配合 xv6 很好
-《操作系统：设计与实现》（Tanenbaum，含 Minix 源码）— 帮助理解小内核结构
-《深入理解计算机系统》（CS:APP）— 与 OS 交叉，尤其 I/O、汇编、内存
- Linux 相关：Robert Love 的《Linux Kernel Development》，以及《Linux 设备驱动程序》（LDD）
- 在线课程：MIT 6.828 (xv6 labs)、OSTEP 课程资料、Coursera / edX 上的 OS 课程
