## 启动栈定义

c

运行

```
// entry.S needs one stack per CPU.
__attribute__ ((aligned (16))) char stack0[4096 * NCPU];
```

1. **`stack0`**：为**每个 CPU 核心**分配启动栈空间
2. **`4096 * NCPU`**：每个 CPU 分配 4KB 栈，总大小 = 4KB × CPU 核心数
3. **`__attribute__ ((aligned (16)))`**：强制栈内存**16 字节对齐**（RISC-V 架构要求）
4. 用途：机器模式下执行 C 代码的栈空间，由 `entry.S` 预先设置。

# 核心函数：start ()

c

运行

```
// entry.S jumps here in machine mode on stack0.
void
start()
```

- **函数属性**：无参数、无返回值（`void`）
- **运行环境**：机器模式（Machine Mode），使用 `stack0` 栈
- **核心功能**：**配置 CPU 寄存器 → 切换到监管者模式 → 跳转到内核 main ()**

### 逐行解析 start () 函数

c

运行

```
// 读取机器状态寄存器 mstatus，保存到变量x
unsigned long x = r_mstatus();
```

- `r_mstatus()`：**读 mstatus 寄存器**（RISC-V 机器状态寄存器）
- `unsigned long`：64 位无符号整型（匹配 RISC-V 64 位寄存器）

c

运行

```
// 清除 MPP 位域（模式位）
x &= ~MSTATUS_MPP_MASK;
```

- `MSTATUS_MPP_MASK`：mstatus 寄存器中**模式位的掩码**
- 作用：清空之前的 CPU 运行模式

c

运行

```
// 设置 MPP 为 Supervisor 模式（监管者模式）
x |= MSTATUS_MPP_S;
```

- `MSTATUS_MPP_S`：代表**监管者模式**
- 作用：配置 `mret` 指令执行后，CPU 切换到**监管者模式**（内核运行模式）

c

运行

```
// 把修改后的 mstatus 写回寄存器
w_mstatus(x);
```

- `w_mstatus()`：**写 mstatus 寄存器**

---

c

运行

```
// 设置机器异常程序计数器 mepc 为 main 函数地址
w_mepc((uint64)main);
```

- `mepc` 寄存器：`mret` 指令执行时，**CPU 会跳转到该寄存器存储的地址**
- `(uint64)main`：将 `main` 函数指针强转为 64 位地址
- 作用：告诉 CPU：退出机器模式后，**直接执行内核主函数 main ()**

---

c

运行

```
// 暂时关闭分页机制（虚拟内存）
w_satp(0);
```

- `satp` 寄存器：RISC-V 页表基址寄存器（控制虚拟内存）
- 写 0 = 关闭分页，CPU 直接使用**物理地址**

---

c

运行

```
// 将所有中断/异常委托给监管者模式
w_medeleg(0xffff);
w_mideleg(0xffff);
```

- `medeleg`：异常委托寄存器
- `mideleg`：中断委托寄存器
- `0xffff`：16 位全 1，代表**所有中断 / 异常都交给监管者模式处理**
- 目的：机器模式不处理中断，全部交给内核（监管者模式）

c

运行

```
// 使能监管者模式的外部中断、时钟中断
w_sie(r_sie() | SIE_SEIE | SIE_STIE);
```

- `sie`：监管者中断使能寄存器
- `SIE_SEIE`：外部中断使能
- `SIE_STIE`：时钟中断使能
- 作用：开启内核需要的核心中断

---

c

运行

```
// 配置物理内存保护：允许监管者模式访问全部物理内存
w_pmpaddr0(0x3fffffffffffffull);
w_pmpcfg0(0xf);
```

- RISC-V 机器模式通过 PMP（物理内存保护）限制监管者模式的内存访问
- `0x3fffffffffffffull`：64 位全地址空间
- `0xf`：允许**读 / 写 / 执行**全部内存
- 作用：给内核**全开内存权限**，无任何限制

运行

// 初始化定时器（生成时钟中断）
timerinit();

c

运行

```
// 读取当前 CPU 核心的 hartid（硬件线程ID）
int id = r_mhartid();
// 把 hartid 写入 tp 寄存器（XV6 用于获取当前CPU编号）
w_tp(id);
```

- `mhartid`：机器模式寄存器，存储**当前 CPU 核心的唯一 ID**
- `tp` 寄存器：RISC-V 线程指针寄存器，XV6 用它保存 CPU ID
- 作用：内核可以通过 `cpuid()` 快速获取当前运行在哪个 CPU 核心

---

c

运行

```
// 执行 mret 指令：切换到监管者模式 + 跳转到 main()
asm volatile("mret");
```

- `asm volatile`：内嵌汇编指令，直接执行硬件指令
- `mret`：机器模式返回指令
    
    1. 从 `mstatus` 恢复 CPU 模式 → 监管者模式
    2. 跳转到 `mepc` 寄存器存储的地址 → `main()` 函数
    
- 这行执行后，**机器模式生命周期结束，内核正式启动！**

# 定时器初始化函数：timerinit ()

c

运行

```
// ask each hart to generate timer interrupts.
void
timerinit()
```

- **函数属性**：无参数、无返回值
- **核心功能**：配置 RISC-V 时钟，**让 CPU 定时产生时钟中断**（内核调度、时间管理的基础）

### 逐行解析 timerinit ()

c

运行

```
// 使能机器模式下的监管者时钟中断
w_mie(r_mie() | MIE_STIE);
```

- `mie`：机器模式中断使能寄存器
- `MIE_STIE`：监管者时钟中断使能位
- 作用：开启时钟中断总开关

c

运行

```
// 使能 sstc 扩展（stimecmp 时钟寄存器）
w_menvcfg(r_menvcfg() | (1L << 63)); 
```

- RISC-V 扩展功能：`sstc` 是标准时钟扩展
- 第 63 位置 1 = 开启该扩展

c

运行

```
// 允许监管者模式访问 time/stimecmp 时钟寄存器
w_mcounteren(r_mcounteren() | 2);
```

- `mcounteren`：控制监管者模式能否访问硬件计数器
- 写 2 = 开放时钟寄存器权限

c

运行

```
// 设置第一次时钟中断：当前时间 + 1000000 个时钟周期后触发
w_stimecmp(r_time() + 1000000);
```

- `r_time()`：读取当前 CPU 时钟周期数
- `stimecmp`：时钟比较寄存器，当 `time` 计数到该值时触发中断
- 作用：**触发第一个时钟中断**，后续中断处理函数会循环更新这个值

# 完整总结

## 1. 代码整体作用

这段代码是 **XV6 内核的启动入口**，完成**机器模式 → 监管者模式**的切换，是操作系统从硬件启动到内核运行的桥梁。

## 2. 两个核心函数

表格

|函数名|参数|返回值|核心功能|
|---|---|---|---|
|`start()`|无|`void`|配置 CPU 寄存器、切换运行模式、初始化内存 / 中断、跳转 main ()|
|`timerinit()`|无|`void`|初始化时钟，开启内核时钟中断|

## 3. 关键流程

1. 汇编 `entry.S` → 跳转到 C 语言 `start()`（机器模式）
2. 配置 CPU 为**监管者模式**
3. 关闭虚拟内存、开放全部内存权限
4. 委托中断、初始化时钟
5. 执行 `mret` → 切换模式 + 跳转到 `main()`
6. 内核正式运行

## 4. 核心知识点

- 运行模式：**机器模式（最高权限）→ 监管者模式（内核权限）**
- 关键指令：`mret`（模式切换 + 跳转）
- 关键寄存器：`mstatus`/`mepc`/`satp`/`stimecmp`
- 核心目的：为 XV6 内核主函数准备好硬件环境
